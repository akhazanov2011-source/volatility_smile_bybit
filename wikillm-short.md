# WikiLLM Short: Volatility Smile Bybit

Проект строит `volatility smile` по опционам Bybit через публичный API V5 и работает только как веб-сервис. Вся Python-логика находится в `app.py`: Flask route'ы, загрузка тикеров, парсинг опционных символов, автоматический выбор страйкового окна, подготовка данных, построение Plotly-графика для `IV`, `theta` или `theta / mark price (%)`, а также фоновое in-memory кеширование готовых графиков. `templates/index.html` — единственный UI-шаблон.

Что важно для LLM:

- В проекте остался один Python-файл: `app.py`.
- CLI-режима, Dash-режима и совместимого `volatility_smile.py` больше нет.
- Ключевая функция проекта — `fetch_and_prepare_data(...)`; она оставляет OTM-крылья и по одной ближайшей ATM/ITM точке.
- `_build_chart_entry(...)` — полный pipeline одной пары `(coin, metric)` до записи кеша (сеть → подготовка → рендер), никогда не бросает исключение наружу.
- Фоновый поток-демон `_background_worker_loop()` раз в 3 минуты обновляет все 18 пар `(coin × metric)` и хранит отрендеренный HTML в `_cache`. Маршрут `/` читает кеш, на промахе/холодном старте синхронно обновляет запрошенную пару через `refresh_combo(...)`.
- Сетевой слой `get_tickers(...)` имеет собственный TTL-кеш 60 с, поэтому фоновый проход делает по одному запросу на монету.
- APScheduler не используется — фоновый цикл реализован на `threading`. `@app.before_first_request` удалён (его нет во Flask 3.x); воркер стартует при импорте модуля через `_ensure_worker_started()`.
- Диапазон страйков больше не задаётся вручную: он рассчитывается автоматически так, чтобы показать 70% ближайших к `spot` страйков.
- В интерфейсе есть переключатель метрики `iv` / `theta` / `theta_pct` и индикатор свежести кеша.
- `build_figure(...)` ожидает уже подготовленную структуру `by_expiry`.
- Поддерживаемые монеты: `BTC`, `ETH`, `SOL`, `XRP`, `DOGE`, `XAUTUSDT`.
- Зависимости: `Flask`, `gunicorn`, `plotly`, `requests`.
- Минимальная проверка после правок: `python3.14 -m py_compile app.py`, затем smoke-import `app`, затем Flask `test_client` (с подменой `_build_chart_entry`, чтобы не ходить в сеть).
